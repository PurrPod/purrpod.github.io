# 常见问题

## 一、环境部署类

**1. 运行需要哪些前置依赖？** 核心依赖为：**uv**（Python 包管理器，一行命令安装）、**Node.js**（提供 `npx`）、**Docker**（沙盒隔离）。

**2. 安装很慢、超时或失败？** 

一般是网络问题，若持续失败，请提前配置国内镜像源（配置 uv 镜像源、Docker 镜像加速）。

**3. 必须安装 Docker 吗？** 

**必须安装**。框架的本地沙盒、Shell 命令执行、系统级读写隔离均强依赖 Docker 容器，无 Docker 无法运行核心功能。

**4. 需要安装 Node.js 吗？** 
**必须安装**。Node.js 提供 `npx` 命令，PurrCat 的 MCP 扩展工具、WebUI 前端均依赖它运行。

**5. purrcat 命令找不到？**
确保在项目根目录下执行。

## 二、运行与使用类

**1. 任务中断后能断点续跑吗？** 支持。框架内置 Checkpoint 持久化机制，每轮对话状态实时落盘。意外退出后重启，只需让 Agent 恢复任务即可自动加载 Checkpoint 继续执行。

**2. Agent 如何记住我的偏好？** 框架内置**轻量级备忘录系统**（Memo 工具 + PurrMemo 本地引擎）。对话中直接告知偏好即可，系统会以极低 token 消耗将其持久化。你也可以通过 Memo 工具主动写入记忆。

**3. API 调用受限怎么办？** 在 `.purrcat/model.json` 的 `api_keys` 列表中添加多个 Key，系统会通过 APIKeyManager 自动进行负载均衡（最少活跃优先）。

**4. Agent 为什么说 Docker 沙盒环境受限？** 大概率是没有启动 Docker 桌面软件，或首次安装后未运行 `purrcat setup` 构建沙盒镜像。

**5. 沙盒机制足够安全吗？** 所有 Shell 执行被强制封锁在 Docker 容器内。对主机的访问通过 `.purrcat/file.json` 定义**严格的白名单机制**，未授权目录绝对不可见。导出操作自动触发 Git 快照防止覆盖灾难。**请遵循最小权限原则，后果自负。**
